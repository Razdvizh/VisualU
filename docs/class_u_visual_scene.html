Detailed Description

Class that loads, visualizes and connects Scenarios in the game.

Visual Scene organizes branches of Scenarios into the non-threaded tree structure, allowing gameplay designers create multiple ways to complete the game. Nodes of the tree are Data Tables, which can have any amount of Scenarios. Nodes are linked by Visual Choice sprite that should reside in the last Scenario. Note that such linking is not enforced: you may place additional Scenarios after the scene with Visual Choice or not add this sprite at all. Visual Scene operates on one node of tree at a time. It loads resources if they weren't already loaded, constructs requested sprites, provides interfaces to the data of the currently active Scenario, and can iterate over them forward and backward or jump to the exhausted, or previously seen, Scenario. Visual Scene is designed to be efficient in both speed and memory, with priority given to the speed. There is no limitations on amount of Visual Sprites or size of the branch, however increase in amount of Scenarios inside Data Table would linearly decrease performance.

◆ ExhaustedScenes

TArray<FScenario*> UVisualScene::ExhaustedScenes
protected

Scenes that has been already seen in the game.

Only the scenes past the currently active scene are exhausted.

Must be used only as a stack data structure, because scene might be restored and removed from the stack. In other words, when player returns back to the scene that was seen previously, or was exhausted, that scene is not considered exhausted anymore untill the player advances forward again.
